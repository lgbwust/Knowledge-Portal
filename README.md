#企业知识门户简介
       企业知识门户(Enterprise Knowledge Portal，EKP)是EIP的一种新表现形式。EKP更关注于企业内部员工和信息内容，它是知识管理系统KM与企业信息门户EIP的结合。一般意义上，EIP更关注于业务过程和内外网的整合。企业知识门户主要包括对企业内部知识的创造、获取、加工、存储、传播和应用的管理等环节。知 识管理是对显性知识和隐性知识的搜集、整理、存储和应用，并使其充分发挥作用的过程。主要体现在以下三个方面： 一是对显性知识的序化，即对显性知识加以序化组织，以便建立知识库，供读者使用；二是对隐性知识的发掘，即强调人是知识管理的核心，图书馆要建立一种创 新、交流、学习和应用知识的环境与激励机制，培养知识型馆员，建立人才库；三是用知识管理的理念指导服务，充分发挥服务的价值和知识的价值。随着知识经济时代的到来，企业在竞争中对机会的把握，越来越取决于组织的智力和知识。特别的，在知识型企业和组织中，知识资产的价值远超过物质资产，是企业的核心竞争力。如何有效的管理组织的智力和知识资产，为企业创造价值，正逐步为各企业所重视。知识管理正是在这样的商业背景驱动下产生的。知识管理的内容，包括对知识的清理、评估、监督、规划、学习、整合、存储、创新、保护、获取、共享、传播等各个环节。
       
       
# 业务需求分析

       企业业务需求分析要明确企业知识门户的用户及他们在组织里的角色，确定门户用户工作所需的业务内容的种类。研究企业业务和服务内容时候，是以调查问卷设计与填写开始的。用户分为企业内部用户和外部用户。分析内部用户要研究组织结构，绘制组织结构图，对组织结构中各个层次进行分析，分析有哪些部门，部门与部门之间有怎样的关系。其次了解组织运作的流程，各个部门、各级员工有哪些具体业务处理。最后根据按照具体处理的业务，在不同的部门，不同的员工，不同的级别之间制定企业知识门户内部用户的角色的分类。对于企业外部管理，研究其供应链上下游的具体情况和运作流程。外部用户可简单的分为供应商、合作伙伴、大客户、代理经销商等等用户角色。

       技术需求分析中，从知识资源管理过程角度分析构建知识门户的关键技术，包括：存储技术——数据仓库、数据集市、知识仓库；分布技术——OLAP技术、数据挖掘、知识挖掘；发布技术——可视化技术；处理与协作技术——Agent技术、知识推送、群件；决策技术——决策支持技术。这些关键技术为企业知识门户的构建提供了强大的技术支持。
